# 🕸️ 知识图谱入口

> 本笔记是 Obsidian Graph View 的使用指南 + 图谱锚点。
> 图谱的连通性靠每篇笔记 frontmatter 的 `up` 字段保证：`笔记 → 领域MOC → 总览 → Home`。

## 图谱层级结构

```
Home（圆心）
 └── MOC-营养学总览（一级地图）
      ├── 9 张领域 MOC（二级地图）
      │    └── 原子知识笔记（叶节点）
      └── MOC-跨学科连接（横向地图，连接不同分支的叶节点）
```

## 推荐 Graph 配置

**全局图谱（Graph View 设置）：**

| 配置项 | 建议 | 目的 |
|--------|------|------|
| 颜色组1 | `path:01_Maps` → 红色 | 一眼看到导航骨架 |
| 颜色组2 | `tag:#type/concept` → 绿色 | 知识主体 |
| 颜色组3 | `tag:#type/case` → 黄色 | 案例 |
| 颜色组4 | `path:10_Journals` → 灰色 | 日志（可过滤掉） |
| 过滤器 | `-path:00_System/Templates` | 模板不参与图谱 |

**局部图谱（每篇笔记内 `Open local graph`）：**
- 深度（Depth）设为 2：看到"邻居的邻居"，最适合发现潜在双链

## 健康度自检（周复习时看一眼）

1. **有没有孤岛？** 灰色孤立节点 = 缺 `up` 或缺双链的笔记，立即补链
2. **MOC 是不是枢纽？** 红色节点应该是度数最高的——如果某篇 concept 笔记比 MOC 连接还多，考虑把它升级为 Layer 3 MOC
3. **跨分支的边多不多？** 不同领域间的连线是跨学科洞见的信号，登记到 [[MOC-跨学科连接]]

## 孤岛笔记动态检测

```dataview
LIST
WHERE !up AND type != "home" AND type != "template"
```
