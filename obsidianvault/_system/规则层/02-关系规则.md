# 02-关系规则

## 1. 只允许 4 种显式关系

| 关系 | 语义 | 典型方向 |
|---|---|---|
| 回应 | A 是对 B（通常是问题）的回答或反应 | 方案/概念/观点 → 问题 |
| 解释 | A 说明 B 为什么成立或如何运作 | 概念 → 观点/方案 |
| 证明 | A 提供了 B 的证据 | 案例 → 概念/观点/方案 |
| 冲突 | A 与 B 存在真实张力，不能同时全取 | 任意 ↔ 任意 |

普通的"相关"用上位/下位/关联链接区表达，**不算显式关系**。

## 2. 书写格式（机器可解析，勿改动格式）

在笔记正文的 `## 显式关系` 小节下，每行一条：

```
- 解释 → [[知识是网络而非仓库]]
- 冲突 → [[知识分层模型]]
```

`validate_vault.py` 用正则 `^- (回应|解释|证明|冲突) → \[\[...\]\]` 统计全库关系，
并与 [[关系索引]] 对账。新增关系后必须重跑脚本。

## 3. 质量要求

- 关系必须给得出理由（写在该行下一行的缩进说明里，或正文中）。
- 冲突关系最有价值，发现真实张力时优先记录。
- 单元可以没有显式关系（罕见），但整库显式关系数 / 单元数 应 ≥ 0.8。
