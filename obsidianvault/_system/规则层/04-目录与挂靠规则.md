# 04-目录与挂靠规则

## 1. 顶层目录（固定，不得增删）

```
00-MOC            导航地图层（主题地图，不存知识只存链接）
01-Domains        领域页（每个领域一页，链到 MOC 与核心单元）
02-Concepts       概念类内容单元
03-Mental Models  思维模型（概念的子类）+ 思维模型总索引
04-People         人物页
05-Books          书籍/权威文献来源页
06-Courses        课程/考证体系页
07-Projects       项目页 + 方案/ + 开放问题/
08-Writing        写作层：选题库、金句库、案例库、故事库、反常识观点库、观点库/、选题装配稿/
09-Resources      资源层：案例/ 等支撑材料
10-Inbox          收件箱（未分类入口，处理后必须清空挂靠）
01-原始素材区      完整副本（只读，不参与图谱校验）
_system           规则层/状态层/模板层/脚本层（系统件，不参与图谱校验）
```

## 2. 挂靠规则

- 每个内容单元必须出现在**至少一个** `00-MOC/` 地图或索引页（思维模型总索引、各库页）的 `[[ ]]` 中。
- 每个 MOC 必须被 [[知识地图]] 引用；[[知识地图]] 是图谱根节点。
- `10-Inbox` 中的文件不算完成态，[[处理状态总览]] 中保持"待处理"直到挂靠完成。

## 3. 校验

`_system/脚本层/validate_vault.py` 检查：frontmatter 完备性、type 合法性、孤立文件 = 0、
每单元出链 ≥1、显式关系格式、与状态层各清单的数字对账。
