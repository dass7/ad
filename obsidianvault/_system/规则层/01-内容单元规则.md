# 01-内容单元规则

## 1. 只允许 5 类内容单元

| type 值 | 中文 | 定义 | 存放目录 |
|---|---|---|---|
| `question` | 问题 | 尚未有定论、驱动后续研究的开放问题 | `07-Projects/开放问题/` |
| `concept` | 概念 | 可复用的概念或思维模型（思维模型加 `subtype: mental-model`） | `02-Concepts/`；思维模型放 `03-Mental Models/` |
| `claim` | 观点 | 有立场、可被赞同或反驳的判断 | `08-Writing/观点库/` |
| `case` | 案例 | 具体的、有过程和结果的实例 | `09-Resources/案例/` |
| `solution` | 方案 | 可执行的方法论、流程、策略 | `07-Projects/方案/` |

不属于这 5 类的信息：要么并入某个单元的正文，要么留在原始素材区，不单独建文件。

## 2. 原子化标准

- 每张笔记只表达一个核心思想，正文 300–800 字。
- 禁止《XX总结》《XX大全》《XX合集》式文件。
- 优先提取**可迁移知识**，不大量复制原文；需要原文时链接 `01-原始素材区` 的副本路径（写在 source 字段）。

## 3. 必备 YAML Frontmatter

```yaml
---
title:
type: question|concept|claim|case|solution
subtype:        # 可选，如 mental-model
domain:
source:         # 指向 01-原始素材区 内的副本相对路径
author:
priority: P1|P2|P3
status: permanent|draft
created:
updated:
tags: []
---
```

## 4. 必备正文结构（concept/claim 类）

核心定义（一句话）→ 底层逻辑（为什么成立）→ 关键案例 → 反例/失效条件 → 应用场景 → 关联知识。

## 5. 必备链接区（所有单元）

- **上位概念** ≥3、**下位概念** ≥3、**关联概念** ≥2（可包含待创建链接，待创建项自动进入 [[待创建概念清单]]）。
- **显式关系**区：见 [[02-关系规则]]。
