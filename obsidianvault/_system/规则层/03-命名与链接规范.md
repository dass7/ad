# 03-命名与链接规范

## 1. 命名

- 黄金法则：**文件名 = 检索时会输入的词**（见观点 [[文件名应等于检索词]]）。
- 概念/观点：直接用概念名，如 `原子笔记.md`、`知识是网络而非仓库.md`。
- 案例：`案例-[主题].md`；方案：`方案-[名称].md`；问题：`问题-[问句].md`。
- MOC：`MOC-[领域名].md`；装配稿：`装配稿-[选题].md`。
- 禁止：数字后缀 `(1)`、无意义名（`笔记1`）、中英混写同义名。

## 2. 链接

- 正文引用一律用 Obsidian 直链 `[[文件名]]`，禁止 `[文字](xxx.md)` 形式的 markdown 内链。
- 外部网址保留标准 URL。
- 指向尚未创建笔记的链接是允许的（生长边界），由脚本汇总到 [[待创建概念清单]]。
- 每个笔记页脚标注挂靠：`*该笔记属于：[[MOC-xxx]]*`。
